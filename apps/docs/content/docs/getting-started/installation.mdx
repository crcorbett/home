# Installation

This guide covers the complete setup process for the monorepo template.

## System Requirements

### Required

- **Bun 1.3.5 or higher** - Primary runtime and package manager
- **Node.js 20 or higher** - Required for some development tools
- **Git 2.30 or higher** - Version control with submodule support

### Recommended

- **VS Code** or **Cursor** - IDE with TypeScript support
- **GitHub CLI** - For workflow automation

## Installing Bun

If you don't have Bun installed, run:

```bash
# macOS and Linux
curl -fsSL https://bun.sh/install | bash

# Windows (PowerShell)
powershell -c "irm bun.sh/install.ps1 | iex"

# Verify installation
bun --version
```

## Clone the Repository

```bash
# Clone with submodules for AI context
git clone --recursive https://github.com/your-org/template.git my-project
cd my-project

# Or if you already cloned, initialize submodules
git submodule update --init --recursive
```

The `--recursive` flag ensures the `.context/` submodules are initialized, which provide source code context for AI assistants.

## Install Dependencies

```bash
bun install
```

This installs all dependencies for the root workspace and all apps/packages. Bun's workspace support automatically links local packages together.

## Verify Installation

Run the following commands to verify everything is set up correctly:

```bash
# Type check all packages
bun run check-types

# Lint all files
bun run check

# Build all packages
bun run build
```

All commands should complete successfully.

## Start Development

```bash
bun run dev
```

This starts Turborepo in development mode, which runs:

- **Web app**: http://localhost:3000
- **Admin app**: http://localhost:3001
- **Docs**: http://localhost:3001

## Environment Variables

Create a `.env` file in the root if you need environment variables:

```bash
# .env
DATABASE_URL=your_database_url
API_KEY=your_api_key
```

Each app can have its own `.env` file as well.

## Troubleshooting

### Bun installation issues

If you encounter permission errors, try:

```bash
# Remove existing installation
rm -rf ~/.bun

# Reinstall
curl -fsSL https://bun.sh/install | bash
```

### Submodule issues

If submodules aren't working:

```bash
# Deinitialize and reinitialize
git submodule deinit -f .
git submodule update --init --recursive
```

### Port conflicts

If ports are already in use, check what's running:

```bash
lsof -i :3000
lsof -i :3001
```

Kill the process or configure different ports in the app's `vite.config.ts`.
